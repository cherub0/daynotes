# DayNotes 分享范围、待办日程与任务列表体验优化设计

## 1. 目标

本轮优化解决三个用户可见问题：分享只能覆盖当前日期、待办无法通过选择器设置日期与时间、编辑器任务列表缺少明确的编辑提示且换行间距过大。完成后，再按仓库中已经合并的编辑器与分享、前端架构、UI 重设计三份设计文档核对历史任务，并修复验证确认的真实缺口。

## 2. 范围

### 2.1 本轮包含

- 分享弹窗选择包含首尾日期的起止范围。
- Markdown ZIP、富文本剪贴板、PDF 和 PNG 图片均支持多日内容。
- 空白日期不出现在分享结果中。
- 待办可设置截止日期和具体时间。
- 截止日期使用应用内日历选择，具体时间使用系统时间选择器。
- 编辑器任务列表增加状态提示、空项引导、紧凑换行和清晰焦点反馈。
- 更新自动化测试、完整 UI 验证和相关说明。
- 对已合并优化任务进行实现与证据核对。

### 2.2 本轮不包含

- 跨日期自动迁移待办或全局任务中心。
- 重复任务、提醒通知、拖拽排序。
- 数据库表迁移；新增待办字段继续存放在现有 JSON 中。
- README 长期路线图中的签名、自动更新、CI 等独立项目。

## 3. 方案选择

采用批量范围查询与现有组件渐进扩展的方案：Rust 新增按日期范围读取笔记的 IPC 命令，前端保留单日导出模型并在其上建立多日集合；待办 JSON 增加可选字段；日历交互从现有实现中抽出可复用能力；编辑器仅调整任务列表相关结构提示和样式。

没有采用前端逐日请求，因为长范围会产生不必要的 IPC 往返且错误处理分散。没有建立时间轴或全局任务系统，因为它会改变当前“每天一篇笔记”的产品边界。

## 4. 分享日期范围

### 4.1 数据接口

Rust 暴露 `get_notes_in_range(start_date, end_date) -> Vec<Note>`：

- 校验两个参数均为 `YYYY-MM-DD`。
- 要求开始日期不晚于结束日期。
- 使用参数化 SQL 的闭区间查询，按日期升序返回。
- 不改变现有 `get_note`、`get_notes_dates` 或数据库结构。

前端在 `src/lib/tauri.ts` 增加对应类型安全封装。分享弹窗打开时以当前日期作为开始和结束日期；范围变化后重新加载。

### 4.2 内容选择规则

- 日期范围包含开始日和结束日。
- `content` 没有可见文本或媒体且 `todos` 为空的记录视为空白并跳过。
- 当前编辑日期落在选择范围内时，使用 React 内存中的 `content` 和 `todos` 覆盖数据库返回值，以包含尚未触发自动保存的编辑。
- 最终条目按日期升序排列。
- 范围中没有可分享内容时禁用导出并显示明确空状态。

### 4.3 导出结构

新增多日导出集合类型，每个条目复用现有 `parseExportDocument(date, html, todos)` 结果：

- Markdown：生成一个合并的 Markdown 文档，每天使用日期一级标题，图片继续打包到同一个 ZIP 的 `images/` 目录，资源名保持唯一。
- 富文本：剪贴板 HTML 按日期分节，纯文本回退包含相同日期顺序。
- PDF：共享预览连续渲染多日内容，每个日期从清晰的日期标题开始；分页算法继续负责避免裁切。
- PNG：生成一张按日期连续排列的长图。
- 单日范围保持现有内容语义。
- 文件名使用 `DayNotes-YYYY-MM-DD`；多日使用 `DayNotes-YYYY-MM-DD_to_YYYY-MM-DD`。

### 4.4 分享弹窗交互

弹窗顶部增加开始日期和结束日期按钮，点击后打开应用内日历。若用户把开始日期选到结束日期之后，则结束日期同步到新的开始日期；反向操作同理。加载时显示整理状态并禁用导出按钮；请求失败时保留弹窗，展示错误和重试入口，不生成残缺文件。

## 5. 待办截止日期与时间

`TodoItem` 增加两个可选字段：

```ts
date?: string; // YYYY-MM-DD
time?: string; // HH:MM
```

旧 JSON 没有 `date` 时继续正常解析。新建待办的 `date` 默认为当前笔记日期，`time` 默认为空。`TodoPanel` 接收 `currentDate`，不改变待办仍归属于当前笔记的存储关系。

进行中的待办显示紧凑日期按钮和时间输入：

- 日期按钮打开复用的日历弹窗，支持键盘方向键、翻月、Enter 选择和 Escape 关闭。
- 时间使用 `input[type="time"]`，由 Windows WebView 提供时间选择体验，同时允许键盘输入和清除。
- 未完成且截止时间早于本地当前时间时显示语义化警示色和“已逾期”辅助文本。
- 已完成组显示已设置的日期时间摘要，但不强化逾期状态。
- 分享和导出中的待办文本包含已设置的截止日期与时间。

## 6. 编辑器任务列表体验

改动严格限制在 `ul[data-type="taskList"]`：

- 列表容器使用轻量内嵌背景和左侧强调线，使任务编辑区域可识别。
- 每个任务项具有紧凑间距、悬停和 `:focus-within` 状态。
- 复选框保持可见，并使用主题变量增强边界和选中状态。
- 空任务段落通过伪元素显示“输入任务内容，按 Enter 新增下一项”。
- 任务项内部 `p` 的上下外边距归零；相邻任务项只保留小间距，避免 Enter 后出现大空隙。
- 工具栏任务列表按钮继续使用 `aria-pressed` 和激活样式，并在激活时提供可读状态说明。
- 普通段落、无序列表和有序列表的排版不改变。

## 7. 组件边界

- `src-tauri/src/lib.rs`：范围查询命令与日期校验。
- `src/lib/tauri.ts`：范围查询 IPC 封装。
- `src/lib/exportDocument.ts`：多日集合、空白判断和各格式合并逻辑。
- `src/components/ShareModal.tsx`：范围状态、加载、错误、重试和导出编排。
- `src/components/ExportPreview.tsx`：多日预览分节。
- `src/components/CalendarPicker.tsx`：在保持主日期选择兼容的前提下提供可复用日历选择能力。
- `src/components/TodoPanel.tsx`：日期时间编辑与逾期展示。
- `src/components/Editor.tsx`、`src/components/editor/EditorToolbar.tsx`：任务列表提示和状态呈现。
- `src/App.tsx`：传递当前日期和当前内存内容，不持有新业务逻辑。

## 8. 错误处理与兼容性

- 无效范围在前后端都被拒绝。
- 新请求开始后，较旧的范围响应不得覆盖最新选择。
- 分享弹窗卸载后不更新状态。
- 某次范围读取失败时不使用部分结果。
- 旧待办 JSON、现有 SQLite 数据和七个既有命令保持兼容。
- 中文 UI、主题变量、键盘可达性和减少动态效果设置继续遵循现有约定。

## 9. 测试与验收

按测试优先顺序覆盖：

- Rust 范围查询的闭区间、顺序、无结果和无效范围。
- 前端范围加载、空白过滤、当前未保存内容覆盖、过期响应保护和重试。
- 四种分享方式的多日顺序、日期标题、资源唯一性和文件名。
- 旧待办 JSON 兼容、新建默认日期、日期选择、时间选择、清除和逾期状态。
- 任务列表空项提示、聚焦状态和段落间距规则。
- 跨月日历键盘操作及焦点返回。
