# DayNotes 前端架构与性能优化设计

## 1. 背景与目标

当前应用功能已经覆盖富文本编辑、待办、分享导出、设置和自动保存，但前端职责集中度较高：

- `src/components/Editor.tsx` 约 825 行，同时负责编辑器初始化、工具栏、代码语言、链接、图片和表格交互。
- `src/App.tsx` 同时负责笔记加载、日期切换、自动保存、设置、弹窗和全局快捷键。
- 分享和设置功能随主界面同步加载。
- 当前生产构建的主 JavaScript 文件约 904 KB，超过 Vite 默认的 500 KB 警告阈值。

第一阶段的目标是在不改变用户可见行为、数据库结构和 Tauri IPC 接口的前提下，降低模块耦合和启动加载体积，为后续 UI、后端与发布优化建立稳定基础。

## 2. 范围

### 2.1 本阶段包含

1. 拆分 `App.tsx` 中的笔记会话与自动保存职责。
2. 拆分 `Editor.tsx` 中的工具栏和编辑器辅助交互职责。
3. 延迟加载分享与设置弹窗。
4. 配置稳定、可解释的 Vite 代码分块策略。
5. 补充与重构边界对应的单元和组件测试。
6. 运行现有完整 UI、全部分享策略、Rust 测试和生产构建验证。

### 2.2 本阶段不包含

- 不重做视觉样式或工具栏布局。
- 不引入 Zustand、Redux 等全局状态库。
- 不修改 SQLite 表结构或已有用户数据。
- 不修改 Tauri 命令名称、参数或返回值。
- 不改变 Markdown、HTML、PDF、图片和 ZIP 的导出语义。
- 不实现系统托盘、自动更新、数据库迁移或凭据安全存储。

这些内容分别进入后续 UI、后端基础和工程发布阶段。

## 3. 实施顺序总览

整体优化按以下阶段依次推进，每个阶段独立设计、实施、验证和提交：

1. 前端架构与性能。
2. UI 体验。
3. 后端基础。
4. 工程发布。

本规格只约束第一阶段。

## 4. 架构设计

### 4.1 笔记会话边界

新增 `useNoteSession` hook，负责当前日期对应的笔记会话：

- 加载笔记和已有笔记日期集合。
- 持有 `content`、`todos` 和 `dirty` 状态。
- 处理日期切换前的最终保存。
- 实现编辑后 2 秒防抖自动保存。
- 提供 `Ctrl+S` 使用的立即保存入口。
- 使用最新值引用避免异步保存读取旧状态。
- 对快速日期切换继续使用“只接受最新请求”的策略，旧请求不得覆盖新日期内容。

`App.tsx` 只编排页面布局、弹窗可见性、设置和 Toast，不再直接实现自动保存时序。

公开接口保持小而明确：hook 接收当前日期和通知回调，返回内容、待办、笔记日期、更新函数、保存函数和日期切换协调函数。具体类型在实施计划中固定，并通过 hook 测试约束。

### 4.2 编辑器边界

保留 `Editor` 作为 Tiptap 生命周期的所有者，避免多个组件直接操作编辑器初始化。将辅助职责拆为：

- `EditorToolbar`：渲染格式按钮并调用传入的编辑器命令。
- `CodeLanguagePicker`：语言列表、当前选项和选择回调。
- `ImageInsertPopover`：图片 URL、文件选择、大小限制和插入触发。
- 继续复用现有 `LinkEditor` 与 `TablePicker`。
- 图片文件读取和校验提取为可独立测试的纯函数，组件只处理交互状态。

编辑器扩展列表、内容同步和 `onChange` 仍由 `Editor` 管理。拆分后任何工具栏按钮的可用条件、快捷操作和生成文档结构必须与当前行为一致。

### 4.3 弹窗延迟加载

`ShareModal` 和 `SettingsModal` 改为 `React.lazy` 动态导入，并由统一的 `Suspense` 回退界面承接加载状态。

要求：

- 主窗口首次渲染不下载分享和设置模块。
- 用户点击入口后显示轻量加载反馈，不出现空白或布局跳动。
- 动态导入失败时显示中文错误提示，并允许关闭弹窗或重试，不导致整个应用白屏。
- 现有弹窗属性接口和操作流程保持不变。

为便于动态导入，两个弹窗模块提供默认导出，同时保留是否需要命名导出的决定权，以不破坏现有测试为准。

### 4.4 Vite 分块策略

在 `vite.config.ts` 中配置可维护的 `manualChunks`：

- React 与 React DOM 归入基础运行时块。
- Tiptap 与 ProseMirror 归入编辑器块。
- lowlight 及其语言定义归入代码高亮块。
- 分享导出相关应用代码主要依靠动态导入形成独立块，不使用脆弱的文件名硬编码。

分块以依赖边界为依据，不为追求极小文件进行过度拆分。生产构建验收指标为：

- 入口主 JavaScript 块低于 500 KB（未压缩、压缩前的 Vite 报告值）。
- 构建不再对入口主块产生超过 500 KB 的警告。
- 初始加载不包含分享与设置页面的主要实现代码。
- 所有块均能由 Tauri 本地资源协议正常加载。

## 5. 数据流与兼容性

页面数据流保持单向：

1. 日期栏请求切换日期。
2. 笔记会话先保存旧日期的脏数据。
3. 会话切换当前日期并加载新笔记。
4. 编辑器和待办面板通过回调更新会话状态。
5. 自动保存只持久化当前会话的最新内容。
6. 分享弹窗接收当前日期、正文和待办的快照。

以下内容必须保持兼容：

- 日期字符串继续使用 `YYYY-MM-DD`。
- 笔记正文继续以 HTML 保存，待办继续以 JSON 保存。
- 2 秒自动保存和 `Ctrl+S` 行为不变。
- 切换日期、退出窗口和组件卸载时不得丢失最后修改。
- 分享导出继续覆盖所有编辑能力和所有分享策略。
- 中文 UI 文案不得无故改变。

## 6. 错误处理

- 笔记加载失败：保留现有内容保护策略并显示 Toast，不把失败响应当作空笔记覆盖状态。
- 自动保存失败：保持脏状态，使下一次自动或手动保存可以重试，并显示中文错误信息。
- 动态模块加载失败：错误边界只影响弹窗区域，主编辑界面继续可用。
- 图片读取失败或超过 10 MB：继续阻止插入并显示明确提示。
- 生产分块加载失败：不得捕获后静默忽略，必须让用户看到可恢复提示。

## 7. 测试与验证

### 7.1 新增自动化测试

- `useNoteSession`：初次加载、防抖保存、立即保存、保存失败重试、快速切换日期、卸载前保存。
- 图片辅助函数：合法图片、超过 10 MB、读取失败。
- 延迟弹窗：未打开时模块内容不存在，打开后显示加载状态并最终渲染。
- 错误边界：动态模块失败时显示中文提示且主界面不崩溃。

### 7.2 必须保持通过的验证

- `npm test`
- `npm run lint`
- `npm run build`
- `npm run verify:complete-ui`
- `npm run verify:rust`
- `npm run verify:evidence`
- `npm run tauri:build`

完整 UI 验证必须继续覆盖全部编辑按钮和 Markdown、HTML、PDF、图片、ZIP 等分享策略，验证证据继续保存到 `verify-output/`。

## 8. 验收标准

第一阶段只有在以下条件全部满足时完成：

1. 用户操作、存储格式和 IPC 接口无破坏性变化。
2. `App.tsx` 不再直接实现笔记自动保存时序。
3. `Editor.tsx` 不再直接渲染全部工具栏和图片插入表单。
4. 分享和设置弹窗仅在首次打开时加载。
5. 入口主 JavaScript 块低于 500 KB，且没有对应的 Vite 大块警告。
6. 新增测试与现有测试全部通过。
7. 完整 UI、分享策略、Rust 和 Tauri 生产构建验证通过。
8. 每个可独立验收的改动使用独立 Git 提交，便于定位和回退。

## 9. 后续阶段入口

第一阶段完成后，按顺序进入：

1. UI 体验设计：设计令牌、基础组件、工具栏分组、保存状态、响应式和无障碍。
2. 后端基础设计：数据库迁移、SMTP 密码安全存储、图片资源文件化。
3. 工程发布设计：CI、自动构建 Release、签名和更新机制。

每个阶段必须重新完成设计批准和实施计划，不在本阶段预埋未使用的抽象。
