# DayNotes 编辑器、分享、单实例与邮箱验证优化设计

## 背景与目标

本轮修复用户测试中发现的七项问题：表格不能选择行列数、图片分享与 Markdown 预览不一致、重复启动产生多个托盘图标、PDF 内容被裁切且依赖系统打印、链接无法添加、Markdown 分享缺少本地图片，以及邮箱配置缺少可验证的测试闭环。

目标是在不改变“每天一篇笔记”核心数据模型的前提下，完善编辑、导出和桌面端体验。界面文案保持简体中文，日期继续使用 `YYYY-MM-DD`，主题样式继续使用 CSS 变量。

## 总体方案

采用“前端负责编辑交互与统一内容规范化，Rust/Tauri 负责可靠的桌面文件生成和系统生命周期”的分层方案：

- Tiptap 前端处理表格选择、表格操作和链接编辑。
- 分享模块先把当前笔记转换为统一的导出文档模型，再由 Markdown ZIP、长图和 PDF 三个输出适配器消费。
- Rust 后端负责 ZIP 写盘、原生分页 PDF、保存对话框衔接、单实例控制和 SMTP 测试。
- 所有导出均显式区分成功、用户取消和失败，不把取消操作显示为错误。

不继续采用浏览器打印生成 PDF，因为它无法满足直接保存和稳定分页；也不把 PDF 全部栅格化为图片，因为这会让文字不可搜索、不可复制并显著增大文件。

## 编辑器设计

### 表格插入与编辑

表格按钮打开一个最大 `20 × 20` 的悬浮网格。鼠标移动时，从左上角到当前格的范围实时高亮，并显示“X 行 × Y 列”；点击后插入对应尺寸的表格，默认首行为表头。键盘离开或点击弹层外部时关闭选择器。

当光标位于表格内时显示紧凑操作栏，提供：

- 在当前行前方或后方插入行；
- 在当前列前方或后方插入列；
- 删除当前行；
- 删除当前列；
- 删除整个表格。

操作直接调用 Tiptap Table 命令。当前状态不可执行的按钮使用 `editor.can()` 禁用，表格外不显示操作栏。

### 链接添加与编辑

链接入口改为编辑器内嵌表单，不再使用 `window.prompt`。打开链接面板前记录当前 ProseMirror 选区，并在确认时恢复选区，避免弹层夺取焦点后链接命令作用于错误位置。

网页链接表单包含显示文字与 URL：

- 有文本选区时，显示文字默认使用选中文字，确认后为该范围设置链接；
- 无文本选区时，显示文字默认使用 URL，确认后插入带链接的新文本；
- 光标位于现有链接内时，加载当前链接并允许修改或取消；
- 未写协议的常见网址自动补全 `https://`；
- 仅接受 Tiptap Link 明确允许的安全协议。

本地文件链接继续通过 Tauri dialog 选择文件，使用正确安装的 dialog 插件 API。Windows 路径转换为编码后的 `file:///` URL；显示文字默认使用文件名。若原生对话框不可用，则给出明确错误，不生成只有文件名而没有有效路径的伪链接。

## 统一导出设计

### 导出文档模型

分享模块只解析一次当前笔记 HTML，形成与渲染方式无关的导出文档模型，包含：日期、标题、段落、标题层级、列表、任务项、引用、代码块、表格、链接、分隔线和图片资源。Markdown、长图和 PDF 适配器从同一模型读取内容，避免三套转换逻辑出现语义偏差。

图片资源区分：

- Base64/Data URL：解码为二进制资源；
- 网络 URL：导出 ZIP 时尝试获取并本地化；获取失败则保留原 URL，并在导出完成提示中说明；
- 本地文件 URL：通过 Tauri 后端在权限范围内读取并复制；无法读取时保留原引用并提示。

资源名统一为 `image-1.<ext>`、`image-2.<ext>`，按正文首次出现顺序稳定编号；重复来源复用同一资源。

### Markdown ZIP

点击“导出 Markdown”后弹出 `.zip` 另存为对话框。默认文件名为 `DayNotes-YYYY-MM-DD.zip`。压缩包结构为：

```text
DayNotes-YYYY-MM-DD.zip
├── YYYY-MM-DD.md
└── images/
    ├── image-1.png
    └── image-2.jpg
```

Markdown 中使用 `images/image-1.png` 形式的相对路径。解压整个目录后，支持本地图片的 Markdown 阅读器可直接显示图片。没有本地化图片时仍生成 ZIP，以保持分享交互一致。

### 图片导出

图片导出不再使用当前只识别部分 Markdown 的 Canvas 行绘制器。它使用与分享预览相同的隐藏渲染容器、导出样式和已加载图片，等待字体与图片资源就绪后生成一张纵向长图。表格、代码块、链接样式和真实图片均保留；不得用 `[图片 N]` 占位文本替代图片。

生成前根据内容宽度设置目标画布，长内容扩展高度。超宽表格和图片在导出样式中限制到内容宽度，并保留等比缩放。

### PDF 导出

点击“导出 PDF”后弹出 `.pdf` 另存为对话框，默认文件名为 `DayNotes-YYYY-MM-DD.pdf`。确认路径后由后端直接生成文件，不打开打印窗口，也不要求使用 `Ctrl+P`。

PDF 使用可搜索、可复制文字的原生排版。版式包含日期标题、正文和待办，支持中文字体嵌入或可移植的中文字体回退。块级元素参与分页计算：普通段落可跨页，标题尽量与后续内容保持同页，表格按行分页，代码块按行分页，图片按比例缩放到页面内容区域内，任何元素不得越过页面边界被裁切。

页面方向按导出模型计算：若存在超宽表格、横向图片，或内容自然宽度超过 A4 纵向可用宽度阈值，则整份文档使用 A4 横向；否则使用 A4 纵向。无论方向如何，单个超宽元素仍缩放到内容区域内。

## 单实例与托盘设计

应用接入兼容当前 Tauri 2 版本的单实例插件，并把它放在所有可能处理启动参数、URL 或文件关联的插件之前初始化。

首次启动的实例创建数据库、调度线程、主窗口和唯一托盘图标。再次运行同一 EXE 时：

1. 第二个进程把启动事件交给首个实例；
2. 首个实例显示并取消最小化主窗口；
3. 主窗口获得焦点；
4. 第二个进程退出，不创建数据库连接、邮件调度器或托盘图标。

关闭主窗口继续执行“隐藏到托盘”；只有托盘菜单“退出”才结束首个实例。单实例行为同时在开发构建和安装版 EXE 中验证。

## 邮箱实际测试设计

设置页增加“发送测试邮件”按钮。测试使用表单中尚未保存的当前值，因此用户可以在提交正式设置前验证配置；测试不要求开启“每日自动发送”。

前端先校验：SMTP 主机非空、端口在 `1..65535`、发件邮箱和收件邮箱格式有效、授权码非空。校验通过后调用独立的 `test_email_settings` Tauri 命令，传入当前 EmailSettings。

后端发送真实邮件：

- 主题：`【DayNotes】邮箱配置测试`；
- 正文：说明这是 DayNotes 测试邮件，并包含本地测试时间；
- 发件人与收件人使用当前表单配置。

SMTP 服务器接受邮件后返回成功；按钮发送期间禁用，避免重复发送。失败按可操作类别转换为中文信息：配置格式错误、DNS/连接失败、连接超时、TLS/证书错误、认证或授权码失败、发件人/收件人被拒绝、服务器其他错误。错误详情保留必要的服务器响应，但不得回显授权码。

本轮继续沿用现有 SQLite 设置存储，不迁移到 Windows Credential Manager。设置页明确说明应填写邮箱服务商生成的 SMTP 授权码，而不是邮箱登录密码。

## 错误处理与用户反馈

- 另存为对话框取消：安静返回，不显示失败 Toast。
- 部分网络图片无法打包：ZIP 仍成功生成，保留网络 URL，并显示包含数量的提醒。
- 本地图片读取失败：指出失败图片，避免生成表面成功但引用必然失效的包。
- PDF 字体或资源加载失败：停止写入并显示具体阶段；不留下半成品文件，或使用临时文件写完后原子替换。
- SMTP 测试失败：显示中文分类和安全的服务端摘要，不保存或记录授权码。
- 单实例回调找不到主窗口：记录错误，但第二实例仍不得创建额外托盘。

## 测试与验收

### 自动测试

- 为导出 HTML 到统一模型增加前端测试，覆盖链接、表格、代码块、任务列表、Base64 图片和重复图片。
- 为 Markdown ZIP 路径和图片稳定命名增加测试，验证 Markdown 中只使用正斜杠相对路径。
- 为 PDF 方向判断、图片缩放和分页边界增加 Rust 单元测试。
- 为邮箱字段校验和 SMTP 错误分类增加测试，验证任何返回信息均不包含授权码。
- 保留并通过现有设置默认值测试。

项目目前没有前端测试框架；实现时引入与 Vite/React 兼容的最小测试配置，仅测试可分离的纯逻辑，不为 UI 大量引入脆弱快照测试。

### 手工验收

- 表格网格能选择并插入 `1 × 1`、`3 × 5` 和 `20 × 20`，表格内所有增删操作正确。
- 选中文字和空选区两种情况下均能添加网页链接；现有链接可编辑和取消；本地文件链接指向所选绝对路径。
- 含 Base64 图片的笔记导出 ZIP 后，解压并打开 Markdown 能显示图片。
- 含标题、代码块、表格、待办和图片的长笔记导出图片后与分享预览一致，不出现图片占位符。
- 纵向普通笔记生成 A4 纵向 PDF；含宽表格或横图的笔记生成 A4 横向 PDF；多页内容和图片均不被裁切。
- 连续启动两个 EXE 后只有一个托盘图标，且首个窗口被唤醒。
- 使用有效 SMTP 授权码可收到测试邮件；错误授权码能得到明确中文错误；自动发送开关关闭时仍可测试。
- 执行 `npm run lint`、`npm run build`、前端单元测试、Rust 单元测试和现有 GUI 验证脚本。

## 范围外事项

- 不在本轮实现跨设备同步、云端图片托管或邮件送达/垃圾箱追踪。
- 不迁移现有数据库中的 Base64 图片为独立附件文件。
- 不引入自定义纸张尺寸；PDF 仅在 A4 纵向与横向之间自动选择。
- 不迁移邮箱授权码到系统凭据库。
